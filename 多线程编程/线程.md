
`线程` 是轻量级的进程（`LWP`：light weight process）。

可以简单理解为线程是程序执行的最小单位，线程是CPU调度和分派的基本单位。

# 线程与进程的区别

- 进程有独立的地址空间，而多个线程共享一个进程的地址空间。
    1. 线程更加节省资源，创建和销毁线程的开销较小。
    2. 线程之间的通信更加高效，因为它们共享内存，而进程之间需要通过IPC（Inter-Process Communication）进行通信。
    3. 每个线程都有自己的寄存器和栈，但共享全局变量和静态变量。
- 线程是程序的最小执行单位，而进程是资源分配的最小单位。

# 子线程的创建

启动程序便会默认创建一个主线程，主线程负责执行程序的入口函数 `main`。

此时就是默认情况下的单线程程序。

## Windows 线程相关的核心函数

头文件一般为 `windows.h`
```cpp
#include <windows.h>
```

**`CreateThread`**：创建线程的函数，返回一个线程句柄。
```cpp
HANDLE CreateThread(
    LPSECURITY_ATTRIBUTES   lpThreadAttributes, // 线程安全属性
    SIZE_T                  dwStackSize,        // 线程堆栈大小
    LPTHREAD_START_ROUTINE  lpStartAddress,     // 线程函数入口地址
    LPVOID                  lpParameter,        // 传递给线程函数的参数
    DWORD                   dwCreationFlags,    // 创建标志
    LPDWORD                 lpThreadId          // 返回线程ID
);

// 使用示例
DWORD WINAPI ThreadFunction(LPVOID lpParam) 
{
    int threadNum = *(int*)lpParam;
    printf("Hello from thread %d!\n", threadNum);
    return 0;
}
```

注：
1. `线程句柄（HANDLE）` 是一个指向线程对象的指针，可以用来操作线程，如等待线程结束、获取线程ID等。
2. `线程ID（DWORD）` 是一个唯一标识线程的整数值，可以用来区分不同的线程。

**`_beginthreadex`**：创建线程的函数，返回一个线程句柄。
```cpp
// 该函数需要包含头文件 <process.h>
HANDLE _beginthreadex(
    void        *security,          // 线程安全属性
    unsigned    stack_size,     // 线程堆栈大小
    unsigned    ( __stdcall *start_address )( void * ), // 线程函数入口地址
    void        *arglist,          // 传递给线程函数的参数
    unsigned    initflag,      // 创建标志
    unsigned    *thrdaddr      // 返回线程ID
);
```

注：
1. `_beginthreadex` 是 C 运行时库提供的线程创建函数，比 `CreateThread` 更安全，因为它会正确地初始化 C 运行时库的线程局部存储（TLS）。
2. 使用 `_beginthreadex` 创建线程时，必须包含 `<process.h>` 头文件。

**`WaitForSingleObject`**：等待指定的线程结束。
```cpp
DWORD WaitForSingleObject(
    HANDLE hHandle,  // 线程句柄
    DWORD  dwMilliseconds // 等待时间，单位为毫秒
);

// 使用示例
HANDLE hThread = CreateThread(...);
WaitForSingleObject(hThread, INFINITE); // 等待线程结束
```

**`CloseHandle`**：关闭线程句柄，释放系统资源。
```cpp
BOOL CloseHandle(
    HANDLE hObject // 线程句柄
);

// 使用示例
HANDLE hThread = CreateThread(...);

CloseHandle(hThread); // 关闭线程句柄
```

## C++ 标准库的线程支持

C++11 引入了 `<thread>` 头文件，提供了跨平台的线程支持。

```cpp
#include <iostream>
#include <thread>

void printMessage()
{
    std::cout  << "What the fuck" << std::endl;
}

int main()
{
    // 创建线程
    std::thread thread1(printMessage);

    return 0;
}
```

### `join` 方法

单纯的创建线程并不会等待线程执行完毕，主线程可能会在子线程执行完之前就结束，导致程序异常终止。
比如出现如下输出：
```
terminate called without an active exception
What the 
```

为了避免这种情况，我们需要在主线程中等待子线程完成，可以使用 `join()` 方法：

```cpp
std::thread thread1(printMessage);
thread1.join();     // 会等待子线程的完成
```

注：
1. `join()` 方法会阻塞调用线程，直到被调用的线程完成执行。
2. 如果主线程在调用 `join()` 之前已经结束，程序会抛出异常。因此，确保在主线程结束之前调用 `join()` 是非常重要的。

## `detach` 方法

如果不需要等待线程完成，而是希望线程在后台运行，可以使用 `detach()` 方法：

```cpp
std::thread thread1(printMessage);
thread1.detach();           // 线程在后台运行，主线程不等待
```

这样，线程会在后台运行，主线程不会等待它完成。

但需要注意，分离的线程在主线程结束后可能会被强制终止，因此要确保分离的线程不会访问已经被销毁的资源。

### 在创建线程时传递参数

如果需要在线程函数中使用参数，可以在创建线程时直接传递参数：

```cpp
void printMessage(int threadNum)
{
    std::cout << "Hello from thread " << threadNum << "!" << std::endl;
}

int main()
{
    int threadNum = 1;
    std::thread thread1(printMessage, threadNum); // 直接传递参数
    thread1.join();
    
    return 0;
}
```

### `joinable` 方法

在调用 `join()` 或 `detach()` 之前，可以使用 `joinable()` 方法检查线程是否可加入或分离：

```cpp
std::thread thread1(printMessage, 1);
if (thread1.joinable()) 
{
    thread1.join();
}
```
