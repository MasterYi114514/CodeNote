# 1. 概述

指针分类：
- 原始指针：如 `int* p`，需要手动管理内存，容易出现内存泄漏和悬空指针等问题。
- 智能指针：如 `std::unique_ptr`、`std::shared_ptr` 和 `std::weak_ptr`。

智能指针是原始指针的封装，会自动管理内存，确保资源在不再使用时被正确释放，避免内存泄漏和悬空指针等问题。
- 智能指针通常在栈上作为局部变量，管理着默认在堆上被动态分配的资源。

# 2. `std::unique_ptr` 独占指针

`std::unique_ptr` 在任何给定时候，只能有一个所有者
- 它不允许复制，但允许移动。
- 当指针超出作用域的时候，资源会被自动释放。
- `所有者`：指向资源的智能指针被称为资源的所有者。`std::unique_ptr` 是独占所有权的智能指针，意味着在任何时候只能有一个 `std::unique_ptr` 拥有某个资源。

**创建`std::unique_ptr`**
```cpp
#include <memory>

// 通过原始指针创建
int *rawPtr = new int(10);
std::unique_ptr<int> uniquePtr1(rawPtr);

// 通过 new 创建
std::unique_ptr<int> uniquePtr2(new int(20));

// 通过 make_unique 创建
std::unique_ptr<int> uniquePtr3 = std::make_unique<int>(30);
```

**通过`get()`方法获取地址/访问原始指针**
```cpp
int *rawPtr = uniquePtr.get();
```

**通过 `->` 和 `*` 操作符访问资源**
```cpp
std::unique_ptr<MyClass> uniquePtr = std::make_unique<MyClass>();
uniquePtr->memberFunction(); // 使用 -> 访问成员函数

std::unique_ptr<int> uniquePtr = std::make_unique<int>(10);
std::cout << *uniquePtr << std::endl; // 输出 10
```

**移动语义**
```cpp
std::unique_ptr<int> uniquePtr1 = std::make_unique<int>(10);
std::unique_ptr<int> uniquePtr2 = std::move(uniquePtr1);        // 转移所有权

if(!uniquePtr1) {
    std::cout << "uniquePtr1 is empty" << std::endl; // uniquePtr1 现在为空
}
```

# 3. `std::shared_ptr` 共享指针

`std::shared_ptr` 允许多个指针共享同一个资源
- 通过引用计数来管理资源的生命周期，当最后一个 `std::shared_ptr` 被销毁时，资源才会被释放。
- 允许复制和移动。

**创建`std::shared_ptr`**
```cpp
#include <memory>

// 通过原始指针创建
int *rawPtr = new int(10);
std::shared_ptr<int> sharedPtr1(rawPtr);

// 通过 new 创建
std::shared_ptr<int> sharedPtr2(new int(20);

// 通过 make_shared 创建
std::shared_ptr<int> sharedPtr3 = std::make_shared<int>(30);
```

**引用计数**
```cpp
std::shared_ptr<int> sharedPtr1 = std::make_shared<int>(10);
std::shared_ptr<int> sharedPtr2 = sharedPtr1;       // 共享资源，引用计数增加
std::cout << "Reference count: " << sharedPtr1.use_count() << std::endl; // 输出 2

sharedPtr2.reset(); // 释放 sharedPtr2 的资源，引用计数减少
std::cout << "Reference count: " << sharedPtr1.use_count() << std::endl; // 输出 1

sharedPtr1.reset(); // 释放 sharedPtr1 的资源，引用计数减少到 0，资源被释放
```

# 4. `std::weak_ptr` 弱引用指针

先了解 `std::shared_ptr` 的循环引用问题：
**循环引用问题**
```cpp
#include <memory>

struct Node {
    std::shared_ptr<Node> next;
};

std::shared_ptr<Node> node1 = std::make_shared<Node>();
std::shared_ptr<Node> node2 = std::make_shared<Node>();
node1->next = node2; // node1 持有 node2
node2->next = node1; // node2 持有 node1，形成循环
```

`std::weak_ptr` 是一种不拥有资源的智能指针，它不会增加引用计数，主要用于解决 `std::shared_ptr` 的循环引用问题。
- `std::weak_ptr` 可以观察 `std::shared_ptr` 管理的资源
  - 不能直接访问资源
  - 不改变引用计数(`use_count()`)
  - 不改变资源的生命周期
- 通过 `lock()` 方法可以尝试获取资源的 `std::shared_ptr`
  - 如果资源已经被释放，则返回一个空的 `std::shared_ptr`。
  - 如果资源仍然存在，则返回一个指向资源的 `std::shared_ptr`。

**使用`std::weak_ptr`**
```cpp
#include <memory>
#include <iostream>

struct Node {
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev; // 使用 weak_ptr 避免循环引用
};

std::shared_ptr<Node> node1 = std::make_shared<Node>();
std::shared_ptr<Node> node2 = std::make_shared<Node>();
node1->next = node2; // node1 持有 node2
node2->prev = node1; // node2 持有 node1 的弱引用
```

**使用`lock()`方法获取资源**
```cpp
std::weak_ptr<int> weakPtr = sharedPtr; // sharedPtr 是一个 std::shared_ptr<int>
if (auto sharedPtrFromWeak = weakPtr.lock()) 
{ 
    // 尝试获取资源, 并进行访问
    std::cout << "Resource is still alive: " << *sharedPtrFromWeak << std::endl;
} 
else 
{
    std::cout << "Resource has been released" << std::endl; // 资源已被释放
}
```
