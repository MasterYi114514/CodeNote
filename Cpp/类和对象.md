# 1. 封装

## 1.1 访问权限

C++中有三种访问权限：`public`、`protected`、`private`

- `public`：公共权限，类内外都可以访问
- `protected`：受保护权限，类内和子类可以访问，类外不可以访问
- `private`：私有权限，只有类内可以访问，类外和子类都不可以访问
  
```cpp
class Base 
{
public:
    int a;          // 公共成员
protected:
    int b;          // 受保护成员
private:
    int c;          // 私有成员
};

int main()
{
    Base obj;
    obj.a = 10;     // 可以访问公共成员
    // obj.b = 20;  // 错误，不能访问受保护成员
    // obj.c = 30;  // 错误，不能访问私有成员
    return 0;
}
```

## 1.2 对private成员的访问

可以通过公有成员函数来访问私有成员：

```cpp
class MyClass 
{
public:
    void setData(int value) 
    {
        data = value;   // 通过公有成员函数设置私有成员
    }

    int getData() const
    {
        return data;    // 通过公有成员函数获取私有成员
    }

private:
    int data;       // 私有成员
};

int main()
{
    MyClass obj;
    obj.setData(42);            // 设置私有成员
    int value = obj.getData();  // 获取私有成员
    return 0;
}
```

可以通过只设置其中的一个函数来实现只读或只写属性：

```cpp
class MyClass 
{
public:
    void setData(int value) 
    {
        data = value;   // 只写属性
    }

    int getData() const
    {
        return data;    // 只读属性
    }
private:
    int data;       // 私有成员
};
```

## 1.3 友元

### 1.3.1 友元函数

友元函数是指定义在类外，但有权访问类的私有成员和受保护成员的函数。使用`friend`关键字声明友元函数。

```cpp
class MyClass 
{
private:
    int data;       // 私有成员
public:
    MyClass(int value) : data(value) {}
    friend void displayData(const MyClass& obj); // 友元函数声明
};

void displayData(const MyClass& obj) 
{
    // 友元函数可以访问私有成员
    std::cout << "Data: " << obj.data << std::endl;
}
```

### 1.3.2 友元类

友元类是指定义在类外，但有权访问另一个类的私有成员和受保护成员的类。使用`friend`关键字声明友元类。

```cpp
class MyClass; // 前向声明

class FriendClass 
{
public:
    void showData(const MyClass& obj);
};

class MyClass 
{
private:
    int data;       // 私有成员
public:
    MyClass(int value) : data(value) {}
    friend class FriendClass; // 友元类声明
};

void FriendClass::showData(const MyClass& obj) 
{
    // 友元类可以访问私有成员
    std::cout << "Data: " << obj.data << std::endl;
}
```

注:
1. 友元关系是单向的，被声明为友元的函数或类可以访问类的私有成员，但类本身不能访问友元的私有成员。
2. 友元关系不是传递的，如果类A是类B的友元，类B是类C的友元，类A并不能访问类C的私有成员。
3. 友元关系不是继承的，子类不能继承父类的友元关系。
4. 成员函数也可以被声明为友元函数, 这样该成员函数可以访问另一个类的私有成员。

# 2. struct和class的区别

cpp中，`struct`和`class`的唯一区别在于默认的访问权限不同：
- `struct`的默认访问权限是`public`
- `class`的默认访问权限是`private`

```cpp
struct MyStruct
{
    int x;          // 默认是public
};

class MyClass
{
    int y;          // 默认是private
};

int main()
{
    MyStruct s;
    s.x = 10;       // 可以访问，因为x是public

    MyClass c;
    // c.y = 20;    // 错误，不能访问，因为y是private
    return 0;
}
```

注:
继承时的默认权限即为访问权限,因此:
- `struct`继承时默认是`public`继承
- `class`继承时默认是`private`继承

# 3. 对象的特性

## 3.1 构造函数和析构函数

- 构造函数：用于初始化对象，在创建对象时自动调用，函数名与类名相同，没有返回值，可以有参数，可以重载。
- 析构函数：用于清理资源，在对象销毁时自动调用，函数名与类名相同，前面加`~`，没有返回值，没有参数，不能重载。

```cpp
class MyClass 
{
public:
    MyClass()          // 构造函数
    {
        // 初始化代码
    }

    ~MyClass()         // 析构函数
    {
        // 清理代码
    }
};
```

注:
1. 如果没有定义构造函数，编译器会自动生成一个默认的无参构造函数。
2. 如果定义了带参数的构造函数，编译器不会再生成默认的无参构造函数。
3. 析构函数不能有参数，也不能重载。
4. 析构函数在对象生命周期结束时自动调用，无需手动调用。

### 3.1.1 构造函数的重载

构造函数的分类:
- 可以按参数分类为：无参构造函数和有参构造函数
- 可以按类型分类为：默认构造函数、拷贝构造函数和移动构造函数

```cpp
class Person
{
public:
    Person()          // 默认构造函数
    {
        name = "Unknown";
        age = 0;
    }

    Person(const std::string& n, int a)  // 有参构造函数
    {
        name = n;
        age = a;
    }

    Person(const Person& other)          // 拷贝构造函数
    {
        name = other.name;
        age = other.age;
    }
private:
    std::string name;
    int age;
}
```

注:
1. 如果没有定义任何构造函数，编译器会自动生成一个默认的无参构造函数。
2. 调用默认无参构造函数时, 不要加括号，例如：`Person p;` 而不是 `Person p();`,否则可能会被当作函数声明。
3. 拷贝构造函数的参数必须是对同类型对象的引用，通常是`const`引用，以防止修改被拷贝对象。

```cpp
int main()
{
    Person p1;                     // 调用默认构造函数
    Person p2("Alice", 30);       // 调用有参构造函数
    Person p3 = p2;               // 调用拷贝构造函数
    // Person(p2);              // 错误，不能这样调用拷贝构造函数
    Person p4(p2);               // 正确，调用拷贝构造函数
    return 0;
}
```

### 3.1.2 cpp的默认构造函数

如果没有定义任何构造函数，编译器会自动生成一个默认的无参构造函数，这个默认构造函数会对类的成员进行默认初始化：
- 对于内置类型（如int、float等），默认初始化不会赋值，成员的值是未定义的（垃圾值）。
- 对于类类型（如std::string等），会调用该类的默认构造函数进行初始化。

在默认情况下,cpp会至少给一个类生成三个函数:
- 默认构造函数(无参, 函数体为空)
- 拷贝构造函数(参数为const引用, 成员逐一拷贝),还有赋值运算符重载
- 析构函数(函数体为空)

#### 浅拷贝与深拷贝

- **浅拷贝**: 直接复制对象的每个成员变量的值。如果成员变量是指针类型，浅拷贝只会复制指针的地址，而不会复制指针所指向的实际数据。这可能导致多个对象共享同一块内存区域，从而引发悬空指针或双重释放等问题。
- **深拷贝**: 不仅复制对象的成员变量的值，还会为指针类型的成员变量分配新的内存，并复制指针所指向的实际数据。这样，每个对象都有自己独立的内存区域，避免了浅拷贝带来的问题。

cpp的默认拷贝构造函数是按位拷贝(浅拷贝), 也就是说把源对象的每个成员逐一拷贝到新对象的对应成员中。

```cpp
class MyClass 
{
public:
    MyClass(int size) 
    {
        data = new int[size];  // 动态分配内存
        this->size = size;
    }

    ~MyClass() 
    {
        delete[] data;         // 释放内存
    }
private:
    int* data;                // 指针成员
    int size;
};

int main()
{
    MyClass obj1(10);         // 创建对象obj1
    MyClass obj2 = obj1;      // 使用默认拷贝构造函数创建obj2（浅拷贝）
    return 0;
}
```

有注意事项如下:
1. 如果定义了有参构造函数，编译器不会再生成默认的无参构造函数, 需要手动定义无参构造函数。
2. 如果定义了拷贝构造函数，编译器不会再生成默认的拷贝构造函数。

## 3.2 初始化列表

初始化列表用于在构造函数中初始化成员变量，语法如下：

```cpp
ClassName::ClassName(参数列表) : 成员1(值1), 成员2(值2), ...
{
    // 构造函数体
}
```

例如:

```cpp
class MyClass
{
public:
    MyClass(int x, int y) : a(x), b(y)  // 使用初始化列表初始化成员变量
    {
        // 构造函数体
    }
private:
    int a;
    int b;
};
```

初始化列表的优点:
1. 提高效率：对于某些类型（如const成员、引用成员、类类型成员），必须使用初始化列表进行初始化，否则会导致编译错误。
2. 避免重复赋值：使用初始化列表可以直接初始化成员变量，避免了在构造函数体内先默认初始化再赋值的过程，提高效率。

## 3.3 对象模型和this指针

cpp中成员变量和成员函数分开存储，成员变量存储在对象的内存空间中，成员函数存储在代码区。

所有的成员函数都有一个隐含的参数`this`，它是一个指向调用该成员函数的对象的指针。通过`this`指针，成员函数可以访问调用对象的成员变量和其他成员函数。

`this`指针本质上是一个指针常量，指向调用该成员函数的对象，不能修改`this`指针的值, 相当于`MyClass* const this;`

```cpp
class MyClass 
{
public:
    MyClass(int value) : data(value) {}

    void display() 
    {
        // 通过this指针访问成员变量
        std::cout << "Data: " << this->data << std::endl;
    }
private:
    int data;
};
```

注:
1. `this`指针只能在非静态成员函数中使用，静态成员函数没有`this`指针。
2. 如果成员函数没有重名的参数或局部变量，可以省略`this->`，直接使用成员变量名。
3. 空指针也可以调用成员函数，但在成员函数中访问成员变量会导致未定义行为(即需要确保函数内不访问成员变量,不调用this指针)。

## 3.4 类作为类的成员

cpp允许类作为另一个类的成员变量：

```cpp
class Address 
{
public:
    Address(const std::string& city, const std::string& street) 
        : city(city), street(street) {}
private:
    std::string city;
    std::string street;
};

class Person 
{
public:
    Person(const std::string& name, int age, const Address& address) 
        : name(name), age(age), address(address) {}
private:
    std::string name;
    int age;
    Address address;  // 类作为成员变量
};

甚至于一个类可以在另一个类的内部定义:

```cpp
class Outer 
{
public:
    class Inner 
    {
    public:
        void display() {}
    };

    void show() 
    {
        Inner innerObj;
        innerObj.display();
    }
};
```

## 3.5 静态成员变量和静态成员函数

静态成员就是成员前面加上`static`关键字，表示该成员属于类本身，而不是类的某个对象。所有对象共享同一个静态成员。

- 静态成员变量：用于存储类的共享数据，所有对象共享同一个静态成员变量。静态成员变量在类内声明, 在类外定义和初始化。
- 静态成员函数：用于操作静态成员变量，可以在没有对象的情况下调用。静态成员函数只能访问静态成员变量，不能访问非静态成员变量。

```cpp
class MyClass 
{
public:
    static int staticVar;               // 静态成员变量

    static void staticFunc()            // 静态成员函数
    {
        // 只能访问静态成员变量
        staticVar++;
    }
};

int MyClass::staticVar = 0;            // 静态成员变量定义和初始化
```

注:
1. 静态成员变量必须在类外初始化，例如：`int MyClass::staticVar = 0;`
2. 静态成员函数不能访问非静态成员变量，因为它们没有`this`指针。
3. 静态成员变量在函数编译时分配内存，生命周期贯穿程序始终。

补充:
在c++11及以后的标准中, 允许部分静态成员变量在类内初始化,需满足以下场景:

1. 静态成员变量是`const`或`constexpr`类型的整型或枚举类型。

```cpp
class MyClass 
{
public:
    static const int staticConstVar = 10;      // 类内初始化
    static constexpr int staticConstExprVar = 20; // 类内初始化
};
```

2. 静态成员变量是`inline`的。

```cpp
class MyClass 
{
public:
    static inline int staticInlineVar = 30;    // 类内初始化
};
```


## 3.6 const关键字

### 3.6.1 const修饰成员函数

可以将成员函数声明为`const`，表示该成员函数不会修改对象的状态（成员变量）。在`const`成员函数中，不能修改任何非`mutable`的成员变量，也不能调用非`const`成员函数。

```cpp
class MyClass 
{
public:
    MyClass(int value) : data(value) {}

    int getData() const                // const成员函数
    {
        mutableData++;              // 可以修改mutable成员变量
        return data;                   // 不能修改data
    }

    void setData(int value) 
    {
        data = value;                  // 可以修改data
    }
private:
    int data;
    mutable int mutableData;  // mutable成员变量可以在const函数中修改
};
```

在函数后面加上`const`，表示该函数不会修改对象的状态, 相当于对`this`指针加上`const`限定符, 即`const MyClass* const this;`

### 3.6.2 const修饰对象

可以将对象声明为`const`，表示该对象的状态不能被修改。对于`const`对象，只能调用`const`成员函数，不能调用非`const`成员函数。

```cpp
int main()
{
    const MyClass obj(10);          // const对象
    int value = obj.getData();      // 可以调用const成员函数
    // obj.setData(20);             // 错误，不能调用非const成员函数
    return 0;
}
```

### 3.6.3 const修饰成员变量

可以将成员变量声明为`const`，表示该成员变量的值在对象的生命周期内不能被修改。`const`成员变量**必须在构造函数的初始化列表中进行初始化**。

```cpp
class MyClass 
{
public:
    MyClass(int value) : constData(value) {} // 在初始化列表中初始化const成员变量

    int getConstData() const 
    {
        return constData; // 不能修改constData
    }
private:
    const int constData; // const成员变量
};
```

# 4. 运算符重载

运算符重载是指为自定义类型（类）定义特定的运算符行为，使得这些运算符可以用于类的对象。通过运算符重载，可以使类的对象像内置类型一样使用运算符。

常见的运算符重载包括：`+`、`-`、`*`、`/`、`==`、`!=`、`<`、`>`、`[]`等。

## 4.1 成员函数方式重载运算符

运算符重载可以通过成员函数的方式实现。成员函数重载运算符时，第一个操作数是调用该成员函数的对象，第二个操作数作为参数传递给成员函数。

```cpp
class MyClass 
{
public:
    MyClass(int value) : data(value) {}

    // 重载+运算符
    MyClass operator+(const MyClass& other) 
    {
        return MyClass(this->data + other.data);
    }

    MyClass operator+(const int value) 
    {
        return MyClass(this->data + value);
    }

    MyClass operator-(const MyClass& other) 
    {
        return MyClass(this->data - other.data);
    }

    int getData() const 
    {
        return data;
    }
private:
    int data;
};
```

注:
1. 成员函数重载运算符时，第一个操作数是调用该成员函数的对象，第二个操作数作为参数传递给成员函数。
2. 不能重载某些运算符，如`::`、`.`、`.*`、`sizeof`等。

## 4.2 非成员函数方式重载运算符

运算符重载也可以通过非成员函数的方式实现。非成员函数重载运算符时，所有操作数都作为参数传递给函数。

```cpp
class MyClass 
{
public:
    MyClass(int value) : data(value) {}

    int getData() const 
    {
        return data;
    }
private:
    int data;
};

// 重载+运算符的非成员函数
MyClass operator+(const MyClass& lhs, const MyClass& rhs) 
{
    return MyClass(lhs.getData() + rhs.getData());
}

int main()
{
    MyClass obj1(10);
    MyClass obj2(20);
    MyClass obj3 = obj1 + obj2;  // 使用重载的+运算符
    return 0;
}
```

注:
1. 非成员函数重载运算符时，所有操作数都作为参数传递给函数。
2. 非成员函数重载运算符时，通常需要访问类的私有成员，可以将该函数声明为类的友元函数。
3. 对于某些运算符（如赋值运算符`=`），通常建议使用成员函数方式重载，以便更好地管理对象的状态。

## 4.3 输出运算符重载

输出运算符`<<`通常通过非成员函数重载，并且需要将该函数声明为类的友元函数，以便访问类的私有成员。

```cpp
#include <iostream>
class MyClass 
{
public:
    MyClass(int value) : data(value) {}

    // 友元函数声明
    friend std::ostream& operator<<(std::ostream& os, const MyClass& obj);
private:
    int data;
};

// 重载<<运算符的非成员函数
std::ostream& operator<<(std::ostream& os, const MyClass& obj) 
{
    os << "Data: " << obj.data;
    return os;
}

int main()
{
    MyClass obj(42);
    std::cout << obj << std::endl;  // 使用重载的<<运算符
    return 0;
}
```

## 4.3 递增和递减运算符重载

递增运算符`++`和递减运算符`--`可以通过成员函数或非成员函数重载。递增和递减运算符有两种形式：前置形式和后置形式。

- 前置形式：`++obj`或`--obj`，返回修改后的对象引用。
- 后置形式：`obj++`或`obj--`，返回修改前的对象值，通常通过一个`int`参数来区分前置和后置形式。

```cpp
class MyClass 
{
public:
    MyClass(int value) : data(value) {}

    // 前置递增运算符重载
    MyClass& operator++() 
    {
        ++data;
        return *this;
    }

    // 后置递增运算符重载
    MyClass operator++(int) 
    {
        MyClass temp = *this; // 保存当前状态
        ++data;               // 递增
        return temp;         // 返回递增前的状态
    }

    int getData() const 
    {
        return data;
    }

private:
    int data;
};
```

注:
1. 前置递增运算符返回对象的引用，以便支持链式调用。
2. 后置递增运算符返回对象的值，因此需要创建一个临时对象来保存递增前的状态。
3. 后置递增运算符通过一个`int`参数来区分前置和后置形式，该参数不使用，仅用于区分。

## 4.4 函数调用运算符重载

函数调用运算符`()`可以通过成员函数重载，使得类的对象可以像函数一样被调用。

```cpp
class MyClass 
{
public:
    MyClass(int value) : data(value) {}

    // 重载函数调用运算符
    int operator()(int increment) 
    {
        data += increment;
        return data;
    }
private:
    int data;
};

int main()
{
    MyClass obj(10);
    int result = obj(5);  // 使用重载的函数调用运算符
    std::cout << "Result: " << result << std::endl; // 输出结果
    return 0;
}
```

注:
1. 函数调用运算符重载允许类的对象像函数一样被调用。
2. 可以根据需要定义函数调用运算符的参数和返回类型。
3. 函数调用运算符重载通常用于实现函数对象（functor）。

# 5. 继承

继承是面向对象编程中的一个重要特性，它允许一个类（派生类）继承另一个类（基类）的成员变量和成员函数，从而实现代码的重用和扩展。

## 5.1 继承的基本语法

常见的继承有三种方式：`public`继承、`protected`继承和`private`继承。

```cpp
class Base 
{
public:
    int publicVar;          // 公共成员
protected:
    int protectedVar;       // 受保护成员
private:
    int privateVar;         // 私有成员
};

class Derived : public Base    // 公共继承
{
    // publicVar 作为 public 成员继承
    // protectedVar 作为 protected 成员继承
    // privateVar 不能被继承
};

class DerivedProtected : protected Base // 受保护继承
{
    // publicVar 作为 protected 成员继承
    // protectedVar 作为 protected 成员继承
    // privateVar 不能被继承
};

class DerivedPrivate : private Base // 私有继承
{
    // publicVar 作为 private 成员继承
    // protectedVar 作为 private 成员继承
    // privateVar 不能被继承
};
```

## 5.2 继承中的访问权限

在继承中，访问权限的规则如下：
- 派生类可以访问基类的`public`和`protected`成员，但不能访问基类的`private`成员。
- 基类的`private`成员只能通过基类的公有或受保护成员函数访问。

```cpp
class Base 
{
public:
    int publicVar;          // 公共成员
protected:
    int protectedVar;       // 受保护成员
private:
    int privateVar;         // 私有成员
public:
    Base() : publicVar(1), protectedVar(2), privateVar(3) {}
    int getPrivateVar() const { return privateVar; } // 通过公有成员函数访问私有成员
};

class Derived : public Base    // 公共继承
{
public:
    void accessMembers() 
    {
        publicVar = 10;          // 可以访问公共成员
        protectedVar = 20;       // 可以访问受保护成员
        // privateVar = 30;      // 错误，不能访问私有成员
        int priv = getPrivateVar(); // 可以通过公有成员函数访问私有成员
    }
};

class DerivedProtected : protected Base // 受保护继承
{
public:
    void accessMembers() 
    {
        publicVar = 10;          // 可以访问公共成员
        protectedVar = 20;       // 可以访问受保护成员
        // privateVar = 30;      // 错误，不能访问私有成员
        int priv = getPrivateVar(); // 可以通过公有成员函数访问私有成员
    }
};

class DerivedPrivate : private Base // 私有继承
{
public:
    void accessMembers() 
    {
        publicVar = 10;          // 可以访问公共成员
        protectedVar = 20;       // 可以访问受保护成员
        // privateVar = 30;      // 错误，不能访问私有成员
        int priv = getPrivateVar(); // 可以通过公有成员函数访问私有成员
    }
};
```

总结:

- 所有继承方式,子类都只能访问父类的`public`和`protected`成员, 父类的`private`成员都不能被子类访问。
- `public`继承: 不改变基类成员的访问权限。
- `protected`继承: 继承来的成员都变为`protected`成员。
- `private`继承: 继承来的成员都变为`private`成员。

注:
1. 基类的`private`成员不能被派生类直接访问, 但是确实存在于派生类, 可以通过基类的公有或受保护成员函数访问。
2. 派生类的成员变量可以与基类的成员变量同名, 这种情况下, 派生类成员变量会隐藏基类成员变量, 可以通过作用域解析运算符访问基类成员变量。
3. 当基类的成员函数有重载时, 派生类(只需要有其中一种重载)会隐藏所有基类的同名成员函数, 可以通过作用域解析运算符访问基类成员函数。

```cpp
class Base 
{
public:
    int var; // 基类成员变量
    Base() : var(1) {}
};

class Derived : public Base 
{
public:
    int var; // 派生类成员变量
    Derived() : var(2) {}
    void showVars() 
    {
        std::cout << "Derived var: " << var << std::endl;          // 访问派生类成员变量
        std::cout << "Base var: " << Base::var << std::endl;      // 访问基类成员变量
    }
};

```

## 5.3 继承的构造与析构顺序

在继承关系中，构造函数和析构函数的调用顺序如下：
- 构造函数调用顺序：先调用基类的构造函数，然后调用派生类的构造函数。
- 析构函数调用顺序：先调用派生类的析构函数，然后调用基类的析构函数。

```cpp
class Base 
{
public:
    Base() 
    {
        std::cout << "Base constructor called" << std::endl;
    }

    ~Base() 
    {
        std::cout << "Base destructor called" << std::endl;
    }
};

class Derived : public Base 
{
public:
    Derived() 
    {
        std::cout << "Derived constructor called" << std::endl;
    }

    ~Derived() 
    {
        std::cout << "Derived destructor called" << std::endl;
    }
};

int main()
{
    Derived obj; // 创建Derived对象
    return 0;
}
```

输出结果:
```
Base constructor called
Derived constructor called
Derived destructor called
Base destructor called
```

注:
如果基类没有默认构造函数，派生类的构造函数必须显式调用基类的有参构造函数。

```cpp
class Base 
{
public:
    Base(int value) 
    {
        std::cout << "Base constructor called with value: " << value << std::endl;
    }
};

class Derived : public Base 
{
public:
    Derived(int value) : Base(value) // 显式调用基类的有参构造函数
    {
        std::cout << "Derived constructor called" << std::endl;
    }
};
```

输出结果:
```
Base constructor called with value: 10
Derived constructor called
```

## 5.4 多继承

C++支持多继承，即一个派生类可以有多个基类。多继承的语法如下：

```cpp
class Base1 
{
public:
    void func1() {}
};

class Base2 
{
public:
    void func2() {}
};

class Derived : public Base1, public Base2 
{
public:
    void func3() {}
};

int main()
{
    Derived obj;
    obj.func1(); // 调用Base1的成员函数
    obj.func2(); // 调用Base2的成员函数
    obj.func3(); // 调用Derived的成员函数
    return 0;
}
```

注:
1. 多继承可能导致命名冲突问题，如果多个基类有同名的成员变量或成员函数，派生类需要通过作用域解析运算符来指定访问哪个基类的成员。
2. 多继承可能导致菱形继承问题，即一个基类被多个派生类继承，然后又有一个类继承了多个该类的派生类, 导致派生类中存在多个基类的副本。

### 5.4.1 菱形继承问题

对于菱形继承问题，可以使用虚继承来解决。虚继承确保只有一个基类的副本被继承。

```cpp
class Base 
{
public:
    void func() {}
};

class Derived1 : virtual public Base 
{
};

class Derived2 : virtual public Base 
{
};

class MostDerived : public Derived1, public Derived2 
{
};

int main()
{
    MostDerived obj;
    obj.func(); // 访问Base的成员函数
    return 0;
}
```

### 5.4.2 多继承的构造函数调用顺序

在多继承中，基类的构造函数按照它们在派生类声明中的顺序被调用。

```cpp
class Base1 
{
public:
    Base1() 
    {
        std::cout << "Base1 constructor called" << std::endl;
    }
};

class Base2 
{
public:
    Base2() 
    {
        std::cout << "Base2 constructor called" << std::endl;
    }
};

class Derived : public Base1, public Base2 
{
public:
    Derived() 
    {
        std::cout << "Derived constructor called" << std::endl;
    }
};

int main()
{
    Derived obj; // 创建Derived对象
    return 0;
}
```

输出结果:

```
Base1 constructor called
Base2 constructor called
Derived constructor called
```

## 5.5 重写与覆盖

在继承关系中，派生类可以重写（覆盖）基类的成员函数，以提供特定的实现。重写是指派生类定义了与基类同名、同参数列表的成员函数。

```cpp
class Base 
{
public:
    void display() 
    {
        std::cout << "Base display" << std::endl;
    }
};

class Derived : public Base 
{
public:
    void display()  // 重写基类的display函数
    {
        std::cout << "Derived display" << std::endl;
    }
};

int main()
{
    Derived obj;
    obj.display(); // 调用Derived的display函数
    return 0;
}
```

输出结果:
```
Derived display
```

注:
1. 重写的成员函数必须与基类的成员函数具有相同的名称、参数列表和返回类型。
2. 如果需要在派生类中调用基类的重写函数，可以使用作用域解析运算符。

```cpp
class Derived : public Base 
{
public:
    void display()  // 重写基类的display函数
    {
        Base::display(); // 调用基类的display函数
        std::cout << "Derived display" << std::endl;
    }
};
```

# 6. 多态

多态是面向对象编程中的一个重要特性，它允许不同的类以相同的接口进行交互，从而实现代码的灵活性和可扩展性。C++中的多态主要通过虚函数实现。

## 6.1 虚函数

虚函数是在基类中使用`virtual`关键字声明的成员函数，允许派生类重写该函数。通过基类的指针或引用调用虚函数时，会根据实际对象的类型调用相应的派生类的函数实现，这就是多态的体现。

```cpp
class Base 
{
public:
    virtual void display()  // 虚函数
    {
        std::cout << "Base display" << std::endl;
    }
};

class Derived : public Base 
{
public:
    void display() override  // 重写虚函数
    {
        std::cout << "Derived display" << std::endl;
    }
};

int main()
{
    Base* ptr = new Derived(); // 基类指针指向派生类对象
    ptr->display();            // 调用派生类的display函数（多态）
    delete ptr;
    return 0;
}
```

注:
1. 虚函数必须在基类中声明为`virtual`，派生类中可以使用`override`关键字来标识重写的虚函数，这有助于编译器检查函数签名是否匹配。
2. 虚函数与非虚函数的重写在内存布局上有所不同，虚函数通过虚函数表（vtable）实现动态绑定。
3. 析构函数通常应声明为虚函数，以确保通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，避免资源泄漏。

## 6.2 纯虚函数和抽象类

一些基类的虚函数并不需要有具体的实现，可以将其声明为纯虚函数，使用`= 0`语法。这种类称为抽象类，**不能实例化**，只能作为基类使用。

```cpp
class Base 
{
public:
    virtual void display() = 0; // 纯虚函数
};

class Derived : public Base 
{
public:
    void display() override  // 实现纯虚函数
    {
        std::cout << "Derived display" << std::endl;
    }
};

int main()
{
    Base* ptr = new Derived(); // 基类指针指向派生类对象
    ptr->display();            // 调用派生类的display函数（多态）
    delete ptr;
    return 0;
}
```

注:
1. 抽象类不能被实例化，只能作为基类使用。
2. 派生类必须实现所有纯虚函数，否则派生类也将成为抽象类。
3. 抽象类可以包含非纯虚函数，这些函数可以有具体的实现。

## 6.3 虚析构函数和纯虚析构函数

析构函数通常应声明为虚函数，以确保通过**基类指针删除派生类对象**时，能够正确调用派生类的析构函数，避免资源泄漏。

```cpp
class Base 
{
public:
    virtual ~Base()  // 虚析构函数
    {
        std::cout << "Base destructor called" << std::endl;
    }
};

class Derived : public Base 
{
public:
    ~Derived() override  // 重写虚析构函数
    {
        std::cout << "Derived destructor called" << std::endl;
    }
};

int main()
{
    Base* ptr = new Derived(); // 基类指针指向派生类对象
    delete ptr;                // 正确调用Derived和Base的析构函数
    return 0;
}
```

输出结果:

```
Derived destructor called
Base destructor called
```

注:
1. 如果基类的析构函数不是虚函数，通过基类指针删除派生类对象时，只会调用基类的析构函数，导致派生类的资源未被释放，可能引发内存泄漏。
2. 纯虚析构函数也是允许的，但必须在类外提供定义，因为析构函数总是会被调用，即使是纯虚析构函数。

```cpp
class Base 
{
public:
    virtual ~Base() = 0; // 纯虚析构函数
};

Base::~Base() 
{
    std::cout << "Base destructor called" << std::endl;
}
```
