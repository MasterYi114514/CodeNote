# 1. 数值数据的表示

数值数据：可以进行数值运算的数据，包括：
- 无符号整数：只能表示非负整数，范围为0到2^n-1（n为位数）。
- 有符号整数：可以表示正数、负数和零，范围为-2^(n-1)到2^(n-1)-1。
- 浮点数：用于表示实数，采用科学计数法表示，包含一个符号位、一个指数部分和一个尾数部分。

## 1.1 进位计数制

进位计数制：一种表示数值的方法，使用一个基数（如2、10、16）和一组符号来表示数值。
- 二进制（基数2）：使用0和1表示数值。
- 十进制（基数10）：使用0到9表示数值。
- 十六进制（基数16）：使用0到9和A到F表示数值。

## 1.2 定点表示和浮点表示

**定点表示**：将数值表示为一个固定的小数点位置，适用于表示整数和小数，但范围有限：
```
.1234       // 小数点前有0位，小数点后有4位
1234.       // 小数点前有4位，小数点后有0位
```

**浮点表示**：将数值表示为一个尾数和一个指数，适用于表示范围较大的数值：
```
123.456 -> .123456 x 10^3
```

这样就实现了用定点数表示浮点数。

## 1.3 定点数的编码表示

**真值**：定点数的实际数值。
**机器数**：定点数在计算机中的二进制表示。

**原码**：直接将数值的二进制表示作为机器数，符号位为0表示正数，符号位为1表示负数。
```
+5 -> 0000 0101
-5 -> 1000 0101

// 0的表示不唯一
0 -> 0000 0000
-0 -> 1000 0000
```

**补码**：将负数的原码取反加1得到补码，符号位为0表示正数，符号位为1表示负数。
```
+5 -> 0000 0101     // 正数的补码与原码相同
-5 -> 1111 1011 (原码取反：1111 1010，加1：1111 1011)

// 0的表示唯一
0 -> 0000 0000

+1 -> 0000 0001
-1 -> 1111 1111 (原码取反：1111 1110，加1：1111 1111)

// 最小的数
1000 0000 -> -128
这里 +128是溢出的，可以认为是：+127 + 1 -> 1000 0000
将其取反：0111 1111，加1：1000 0000 -> -128
这里可以发现：二者是相同的，说明了补码的循环性质。
```

# 2. 整数的表示

## 2.1 无符号整数和有符号整数

**无符号整数**：直接将数值的二进制表示作为机器数，范围为0到2^n-1。
```
+5 -> 0000 0101
```

**有符号整数**：使用补码表示，范围为-2^(n-1)到2^(n-1)-1。
```
+5 -> 0000 0101
-5 -> 1111 1011 (补码表示)
```

## 2.2 C语言中无符号整数和有符号整数的表示

1. 内容不变：类型转换前后的机器数不变，即存储的0/1位不变。
2. 解释不同：无符号整数和有符号整数的解释不同，无符号整数将机器数解释为0到2^n-1的范围，有符号整数将机器数解释为-2^(n-1)到2^(n-1)-1的范围。

```c
int x = -1;                 // 0xFFFFFFFF
unsigned u = 2147483647;    // 0x7FFFFFFF，最高位字节为：0111 1111
printf("x = %d = %u\n", x, x);
printf("u = %d = %u\n", u, u);
```

有以下输出：
```
x = -1 = 4294967295
u = 2147483647 = 2147483647
```

# 3. 浮点数的表示

依照 `IEEE 754` 标准，浮点数由一个符号位、一个指数部分和一个尾数部分组成。
- 符号位：0表示正数，1表示负数。
- 指数部分：使用偏移量表示法，存储的值为实际指数加上一个偏移量（如127）。
- 尾数部分：存储有效数字的二进制表示，通常使用隐含的1来表示。
