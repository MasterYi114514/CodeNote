库文件分类：静态库和动态库，其中动态库也叫共享库

# 后缀名

| 操作系统    | 静态库的后缀名 | 动态库的后缀名  |
| ------- | ------- | -------- |
| Windows | `.lib`  | `.dll`   |
| Linux   | `.a`    | `.so`    |
| macOS   | `.a`    | `.dylib` |

# 静态库（Static Library)

**编译时链接，运行时独立**：在项目编译（构建）阶段，编译器会把静态库中的所有相关代码，**完整地复制**到可执行文件（`.exe`/`a.out`）中，生成最终可执行文件后，**静态库文件就不再需要了**，程序运行时完全独立，不依赖任何外部库文件

# 动态库（Dynamic/Shared Library)

**编译时仅链接接口，运行时才加载**：项目编译阶段，编译器不会复制动态库的代码，**只是记录下需要使用的动态库名称和接口信息**；当程序运行时，操作系统才会将动态库加载到内存中，供程序调用，**程序运行时必须依赖动态库文件存在**。

注：Windows系统下 `.dll` 是运行时真正需要的动态库，用导入库 `.lib` 在编译阶段使用，告诉编译器**动态库中的可用接口**，在生成可执行文件后 `.lib` 导入库就可以丢弃。
