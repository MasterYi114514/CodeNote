# OSI 网络模型分层

- `应用层`：提供网络服务，协议有：HTTP、FTP、SMTP
- `表示层`：负责数据的表示和转换，协议有：ASCII、JPEG、MPEG
- `会话层`：负责建立、管理和终止会话，协议有：RPC、NetBIOS
- `传输层`：协议有：TCP、UDP
- `网络层`：负责网络地址的分配和路由选择，协议：IP
- `数据链路层`：负责传输数据帧，如以太网，协议：ATM、PPP
- `物理层`：负责传输比特流，如电缆、光纤，协议：RS-232、USB

该分层可以简化为 `tcp/ip` 四层模型：
- `应用层`：包括七层模型中的 `应用层`、`表示层` 和 `会话层`
- `传输层`：对应七层模型中的 `传输层`
- `网络层互联层`：对应七层模型中的 `网络层`
- `网络接口层`：对应七层模型中的数据 `链路层` 和 `物理层`

## 字节序

- `大端字节序`：高位字节存储在低地址，低位字节存储在高地址。
- `小端字节序`：低位字节存储在低地址，高位字节存储在高地址。

转换函数：
```c
#include <arpa/inet.h>

// 将主机字节序转换为网络字节序
uint16_t htons(uint16_t hostshort); 

// 将网络字节序转换为主机字节序
uint16_t ntohs(uint16_t netshort);  

// 将主机字节序转换为网络字节序
uint32_t htonl(uint32_t hostlong);

// 将网络字节序转换为主机字节序
uint32_t ntohl(uint32_t netlong);  
```

## IP 地址转换

```c
// 下面的两个函数适用于 linux 系统
#include <arpa/inet.h>

// 将点分十进制字符串转换为网络字节序的二进制形式
int inet_pton(int af, const char *src, void *dst);
/*
参数说明：
- af：地址族，AF_INET表示IPv4，AF_INET6表示IPv6。
- src：输入的点分十进制字符串。
- dst：输出的网络字节序的二进制形式，必须足够大以存储结果。

返回值：
- 成功：返回1。
- 输入无效：返回0。
- 其他错误：返回-1，并设置errno。

erron值说明：
- EAFNOSUPPORT：地址族不受支持。
- EINVAL：输入的字符串无效。
*/

// 将网络字节序的二进制形式转换为点分十进制字符串
const char *inet_ntop(int af, const void *src, char *dst, socklen_t size);
/*
参数说明：
- af：地址族，AF_INET表示IPv4，AF_INET6表示IPv6。
- src：输入的网络字节序的二进制形式。
- dst：输出的点分十进制字符串，必须足够大以存储结果。
- size：dst缓冲区的大小。

返回值：
- 成功：返回dst指针。
- 输入无效：返回NULL，并设置errno。

erron值说明：
- EAFNOSUPPORT：地址族不受支持。
- ENOSPC：dst缓冲区太小，无法存储结果。
*/
```

## `sockaddr_in` 结构体

```c
struct sockaddr
{
    sa_family_t sa_family;      // 地址族，AF_INET表示IPv4
    char sa_data[14];           // 地址数据，端口 + IP地址 + 填充
}

typedef unsigned short uint16_t;
typedef unsigned int uint32_t;
typedef uint16_t in_port_t;         // 端口号，使用网络字节序
typedef uint32_t in_addr_t;         // IPv4地址，使用网络字节序
typedef unsigned short int sa_family_t;    // 地址族类型

struct in_addr
{
    in_addr_t s_addr;           // IPv4地址，使用网络字节序
};

// sizeof(struct sockaddr) == 16 == sizeof(struct sockaddr_in)
struct sockaddr_in {
    sa_family_t sin_family;         // 地址族，AF_INET表示IPv4
    in_port_t sin_port;             // 端口号，使用网络字节序
    struct in_addr sin_addr;        // IPv4地址，使用网络字节序
    char sin_zero[8];               // 填充字段，保持结构体大小为16字节
};
```

# `TCP` 通信流程

- `面向连接`：通信双方在数据传输前需要建立连接，使用三次握手（three-way handshake）来建立连接，使用四次挥手（four-way handshake）来关闭连接。
  - `三次握手`：客户端发送SYN包，服务器回复SYN-ACK包，客户端再发送ACK包，连接建立成功。
  - `四次挥手`：客户端发送FIN包，服务器回复ACK包，服务器再发送FIN包，客户端回复ACK包，连接关闭成功。
- `可靠传输`：通过序列号、确认号、重传机制等保证数据的可靠传输。
- `流量控制`：通过滑动窗口机制控制数据的发送速率，避免网络拥塞。
- `全双工通信`：通信双方可以同时发送和接收数据。
