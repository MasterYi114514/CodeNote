
# 1. SRAM(static random access memory)

SRAM = 静态随机访问存储器。
1. 靠晶体管导通/截止来存储数据，断电后数据会丢失。
2. 不像DRAM需要定期刷新，SRAM不需要刷新，访问速度更快。

与DRAM进行对比：

| 特点 | SRAM                      | DRAM               |
| ---- | ------------------------- | ------------------ |
| 速度 | 极快                      | 较慢               |
| 功耗 | 较高                      | 较低               |
| 成本 | 较高                      | 较低               |
| 用途 | CPU缓存、寄存器等高速存储 | 主内存等大容量存储 |

# 2. Cache

CPU 运算速度远快于内存访问速度，为了提高性能，CPU引入了 `Cache`（高速缓存）来存储最近使用的数据和指令。
`Cache` 位于 CPU 和内存之间，访问速度比内存快得多，但容量较小，用 `SRAM` 为主存储介质。

## 2.1 Cache的层次结构

现代计算机通常采用多级Cache结构，常见的有三级Cache：
1. L1 Cache：一级缓存，位于CPU核心内部，速度最快，容量最小，通常为32KB。
2. L2 Cache：二级缓存，位于CPU核心内部或外部，速度较快，容量较大，通常为256KB。
3. L3 Cache：三级缓存，位于CPU核心外部，速度较慢，容量较大，通常为8MB。

层级访问顺序：
```
CPU -> L1 Cache -> L2 Cache -> L3 Cache -> 主存（DRAM）
```

## 2.2 局部性原理与 Cache 命中

程序在执行过程中具有 `局部性原理`，即访问的数据和指令往往集中在某个范围内。
1. 时间局部性：如果某个数据被访问，那么在不久的将来它很可能会被再次访问。
2. 空间局部性：如果某个数据被访问，那么在不久的将来它附近的数据很可能会被访问。

因此，`Cache` 会一次性加载一块连续数据（Cache Line），而非单个字节。

当CPU访问数据时，首先检查 `Cache` 中是否存在该数据：
- 如果存在，称为 `Cache命中`，CPU直接从 `Cache` 中读取数据，访问速度快。
- 如果不存在，称为 `Cache未命中`，CPU需要从主存中加载数据到 `Cache` 中，然后再从 `Cache` 中读取数据，访问速度较慢。

注：
以下代码的两个函数 `copy1` 和 `copy2` 都是将一个二维数组从 `src` 复制到 `dst`，二者时间复杂度相同，但由于访问模式不同，`copy1` 的 `Cache` 命中率更高，性能更好。
```c
#define N 2048
void copy1(int src[N][N], int dst[N][N])
{
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            dst[i][j] = src[i][j];      // 按行访问，Cache命中率高
        }
    }
}

void copy2(int src[N][N], int dst[N][N])
{
    for (int j = 0; j < N; j++) {
        for (int i = 0; i < N; i++) {
            dst[i][j] = src[i][j];      // 按列访问，Cache命中率低
        }
    }
}
```
