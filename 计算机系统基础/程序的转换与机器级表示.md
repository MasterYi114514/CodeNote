
# 1. 程序转换概述

## 1.1 指令

计算机中的指令有：微指令、机器指令和伪（宏）指令
- **微指令**：CPU内部执行的基本操作，如控制信号的设置、寄存器的读写等，通常不可见。
- **机器指令**：CPU能够直接执行的指令，由操作码和操作数组成，存储在内存中。
- **伪（宏）指令**：汇编语言中的指令，不能直接被CPU执行，需要被汇编器翻译成一条或多条机器指令。

# 2. IA-32 指令系统

IA-32（Intel Architecture, 32-bit）是Intel公司推出的一种32位的指令系统，广泛应用于个人计算机中。
  - IA-32指令系统包含了丰富的指令集，支持多种数据类型和寻址模式，适用于各种应用场景。

## 2.1 寄存器体系

**通用寄存器**

| 32位全名 | 16位别名 | 8位别名(高/低) | 用途 |
| --------- | -------- | --------------- | ---- |
| EAX       | AX       | AH/AL           | 累加器，常用于算术运算和函数返回值 |
| EBX       | BX       | BH/BL           | 基址寄存器，常用于存储数据的基地址 |
| ECX       | CX       | CH/CL           | 计数器，常用于循环和字符串操作 |
| EDX       | DX       | DH/DL           | 数据寄存器，常用于输入输出操作 |
| ESI       | SI       | -               | 源索引寄存器，常用于字符串操作 |
| EDI       | DI       | -               | 目的索引寄存器，常用于字符串操作 |
| EBP       | BP       | -               | 基指针寄存器，常用于函数调用和栈帧管理 |

这里的前缀 `E` 表示扩展（Extended），是从16位寄存器扩展到32位寄存器的标志。

**段寄存器**

- `CS` ：代码段寄存器，指向当前执行的代码所在的内存段。
- `DS` ：数据段寄存器，指向程序使用的数据所在的内存段。
- `SS` ：栈段寄存器，指向程序使用的栈所在的内存段。
- `ES` ：附加段寄存器，常用于字符串操作。
- `FS` 和 `GS` ：额外段寄存器，通常用于操作系统和线程局部存储。

**指令指针与标志寄存器**

- `EIP` ：指令指针寄存器，指向下一条要执行的指令的地址。
- `EFLAGS` ：标志寄存器，包含了各种状态标志
  - 零标志（ZF）：当结果为零时置为1，否则为0。
  - 符号标志（SF）：当结果为负数时置为1，否则为0。
  - 进位标志（CF）：当运算产生进位或借位时置为1，否则为0。
  - 溢出标志（OF）：当有符号运算结果超出表示范围时置为1，否则为0。
  - 中断标志（IF）：当允许中断时置为1，否则为0。

## 2.2 IA-32的寻址方式

**立即数寻址**

操作数直接写在指令中。
```
mov eax, 0x12345678     ; 将立即数0x12345678加载到eax寄存器中
```

**寄存器寻址**

操作数在寄存器中。
```
mov eax, ebx            ; 将ebx寄存器的值复制到eax寄存器中
```

**内存寻址**

内存地址的计算公式：
有效地址 EA = 基址 + 变址 * 比例因子 + 偏移量

```
; 直接寻址（偏移量）
mov eax, [0x12345678]  ; 从内存地址0x12345678加载数据到eax寄存器中

; 基址寄存器 + 偏移量
mov eax, [ebx+0x12345678]  ; 从内存地址ebx+0x12345678加载数据到eax寄存器中

; 基址寄存器 + 变址寄存器 * 比例因子
mov eax, [ebx + esi*4]     ; 数组访问，假设每个元素占4字节

; 偏移量[寄存器*比例因子]
mov al, 5[ebx*4]           ; 从内存地址5 + ebx*4加载一个字节到al寄存器中
```

写法为：[BR + IR * F + V] 等效为：
- V[BR][IR * F]
- V[IR * F][BR]
- V[BR + IR * F]

## 2.3 IA-32补充

**push/pop指令**
- `push` 指令将数据压入栈中，栈顶指针 `ESP` 向下移动。
- `pop` 指令将数据从栈中弹出，栈顶指针 `ESP` 向上移动。

```assembly
push eax    ; 将eax寄存器的值压入栈中
pop ebx     ; 从栈顶弹出一个值到ebx寄存器
```

**lea指令**
- `lea`（Load Effective Address）指令用于计算内存地址，并将结果加载到寄存器中，而不访问内存。

```assembly
lea eax, [ebx + esi*4]  ; 计算内存地址ebx + esi*4，并将结果加载到eax寄存器中
```

**控制转移指令**

- `jmp` ：无条件跳转到指定地址。
- `je` / `jne` ：根据零标志（ZF）进行条件跳转（等于/不等于）。
- `call` ：调用函数，跳转到函数地址并将返回地址压入栈中。
- `ret` ：返回函数，从栈中弹出返回地址并跳转到该地址。

# 3. 常用指令

## 3.1 数据传送指令

**一般数据传送指令**

- `mov` ：将数据从源操作数复制到目的操作数。
- `movsx` ：将源操作数符号扩展后复制到目的操作数。 
- `movzx` ：将源操作数零扩展后复制到目的操作数。

```
; mov 要求源操作数和目的操作数位数一致
mov eax, ebx      ; 将ebx寄存器的值复制到eax寄存器中
mov al, bl        ; 将bl寄存器的值复制到al寄存器中

; movsx 将小数据类型扩展到大数据类型，高位补充符号位
movsx eax, bl     ; 将bl寄存器的值符号扩展后复制到eax寄存器中
; 假设 bl = 0xFF，则 eax = 0xFFFFFFFF
; 假设 bl = 0x7F，则 eax = 0x0000007F

; movzx 将小数据类型扩展到大数据类型，高位补充0
movzx eax, bl     ; 将bl寄存器的值零扩展后复制到eax寄存器中
; 假设 bl = 0xFF，则 eax = 0x000000FF
; 假设 bl = 0x7F，则 eax = 0x0000007F
```

**堆栈操作指令**

- `push` ：将数据压入栈中，栈顶指针 `ESP` 向下移动。
- `pop` ：将数据从栈中弹出，栈顶指针 `ESP` 向上移动。
- `pusha` ：将所有16位通用寄存器压入栈中。
- `popa` ：将所有16位通用寄存器从栈中弹出
- `pushad` ：将所有32位通用寄存器压入栈中。
- `popad` ：将所有32位通用寄存器从栈中弹出。

```assembly
; push 执行两步动作
; ESP = ESP - 4
; [ESP] = 数据
push eax    ; 将eax寄存器的值压入栈中

; pop 执行两步动作
; 数据 = [ESP]
; ESP = ESP + 4
pop ebx     ; 从栈顶弹出一个值到ebx寄存器

; pusha 将所有16位通用寄存器压入栈中，顺序为：AX, CX, DX, BX, original SP, BP, SI, DI
; popa 将所有16位通用寄存器从栈中弹出，顺序与pusha相反
pusha        ; 将所有16位通用寄存器压入栈中
popa         ; 将所有16位通用寄存器从栈中弹出

; pushad 将所有32位通用寄存器压入栈中，顺序为：EAX, ECX, EDX, EBX, original ESP, EBP, ESI, EDI
; popad 将所有32位通用寄存器从栈中弹出，顺序与pushad相反
pushad       ; 将所有32位通用寄存器压入栈中
popad        ; 将所有32位通用寄存器从栈中弹出
```

**标志寄存器传送指令**

- `pushf` ：将16位标志寄存器的值压入栈中。
- `popf` ：将16位标志寄存器的值从栈中弹出。
- `pushfd` ：将32位标志寄存器的值压入栈中。
- `popfd` ：将32位标志寄存器的值从栈中弹出。
- `lahf` ：将标志寄存器的低8位加载到AH寄存器中。
- `sahf` ：将AH寄存器的值存储到标志寄存器的低8位中。

```assembly
mov al, 5
add al, 3       ; 这会改变标志位（ZF、CF等）

pushfd          ; 把当前所有标志位 保存进栈
add al, 10      ; 这条指令又会改变标志位
popfd           ; 把栈里保存的标志 恢复回去
```

**地址传送指令**

- `lea` ：计算内存地址，并将结果加载到寄存器中，而不访问内存。

```assembly
mov ebx, 10      ; ebx = 10
mov ecx, 3       ; ecx = 3

lea eax, [ebx + ecx*4 + 5]
; 计算内存地址：10 + 3*4 + 5 = 27
; 将结果27加载到eax寄存器中
```

**带条件的数据传送指令**

- `cmove` ：当零标志（ZF）为1时，执行数据传送。
- `cmovne` ：当零标志（ZF）为0时，执行数据传送。
- `cmova` ：当无符号比较结果为大于时，执行数据传送。
- `cmovb` ：当无符号比较结果为小于时，执行数据传送。
- `cmovae` ：当无符号比较结果为大于或等于时，执行数据传送。
- `cmovbe` ：当无符号比较结果为小于或等于时，执行数据传送。
- `cmovg` ：当有符号比较结果为大于时，执行数据传送。
- `cmovl` ：当有符号比较结果为小于时，执行数据传送。
- `cmovge` ：当有符号比较结果为大于或等于时，执行数据传送。
- `cmovle` ：当有符号比较结果为小于或等于时，执行数据传送。

`above` 和 `below` 是无符号比较的术语，分别对应于 `greater` 和 `less` 的无符号版本。

```assembly
cmp eax, ebx     ; 比较eax和ebx，设置标志位
cmove ecx, edx   ; 如果eax == ebx（ZF=1），将edx的值复制到ecx中
cmovne ecx, edx  ; 如果eax != ebx（ZF=0），将edx的值复制到ecx中
cmova ecx, edx   ; 如果eax > ebx（无符号比较），将edx的值复制到ecx中
```

**带条件的跳跃指令**

- `je` / `jne` ：根据零标志（ZF）进行条件跳转（等于/不等于）。
- `ja` / `jb` ：根据无符号比较结果进行条件跳转（大于/小于）。
- `jae` / `jbe` ：根据无符号比较结果进行条件跳转（大于或等于/小于或等于）。
- `jg` / `jl` ：根据有符号比较结果进行条件跳转（大于/小于）。
- `jge` / `jle` ：根据有符号比较结果进行条件跳转（大于或等于/小于或等于）。

```assembly
cmp eax, ebx     ; 比较eax和ebx，设置标志位
je equal         ; 如果eax == ebx（ZF=1），跳转到equal标签
jne not_equal     ; 如果eax != ebx（ZF=0），跳转到not_equal标签
ja above         ; 如果eax > ebx（无符号比较），跳转到above标签
jb below         ; 如果eax < ebx（无符号比较），跳转到below
```

## 3.2 算术运算指令

**加法指令**

- `inc` ：将寄存器或内存中的值加1。
- `add` ：将两个操作数相加，并将结果存储在目的操作数中。
- `adc` ：带进位加法，将两个操作数和进位标志相加，并将结果存储在目的操作数中。

```assembly
inc eax       ; 将eax寄存器的值加1，不影响 CF 标志位
add eax, ebx  ; eax = eax + ebx，设置标志位（ZF、CF等）
adc eax, 5    ; eax = eax + 5 + CF，设置标志位（ZF、CF等）
```

**减法指令**

- `dec` ：将寄存器或内存中的值减1。
- `neg` ：将寄存器或内存中的值取反（相当于0减去该值）。
- `sub` ：将目的操作数减去源操作数，并将结果存储
- `sbb` ：带借位减法，将目的操作数减去源操作数和借位标志，并将结果存储在目的操作数中。
- `cmp` ：比较两个操作数，设置标志位，但不存储结果。

```assembly
dec eax         ; 将eax寄存器的值减1，不影响 CF 标志位
neg eax         ; 将eax寄存器的值取反（eax = 0 - eax），设置标志位（ZF、CF等）
sub eax, ebx    ; eax = eax - ebx，设置标志位（ZF、CF等）
sbb eax, 5      ; eax = eax - 5 - CF，设置标志位（ZF、CF等）
cmp eax, ebx    ; 比较eax和ebx，设置标志位（ZF、CF等），但不存储结果
```

**乘法指令**

- `mul` ：无符号乘法，将源操作数与累加器寄存器相乘，并将结果存储在累加器寄存器中。
- `imul` ：有符号乘法，将源操作数与累加器寄存器相乘，并将结果存储在累加器寄存器中。

```assembly
mul ebx        ; 无符号乘法，eax = eax * ebx，设置标志位（ZF、CF等）
imul ebx       ; 有符号乘法，eax = eax * ebx，设置标志位（ZF、CF等）
```

**除法指令**

- `div` ：无符号除法，将累加器寄存器中的值除以源操作数，并将商存储在累加器寄存器中，余数存储在数据寄存器中。
- `idiv` ：有符号除法，将累加器寄存器中的值除以源操作数，并将商存储在累加器寄存器中，余数存储在数据寄存器中。

```assembly
div ebx        ; 无符号除法，eax = eax / ebx，edx = eax % ebx，设置标志位（ZF、CF等）
idiv ebx       ; 有符号除法，eax = eax / ebx，edx = eax % ebx，设置标志位（ZF、CF等）
```

**符号扩展指令**

- `cbw` ：将AL寄存器中的8位值符号扩展到AX寄存器中。
- `cwd` ：将AX寄存器中的16位值符号扩展到DX寄存器中。
- `cwde` ：将AX寄存器中的16位值符号扩展到EAX寄存器中。
- `cdq` ：将EAX寄存器中的32位值符号扩展到EDX寄存器中。

```assembly
cbw             ; AL -> AX，符号扩展，将字节扩展为字
cwd             ; AX -> DX，符号扩展，将字扩展为双字
cwde            ; AX -> EAX，符号扩展，将字扩展为双字
cdq             ; EAX -> EDX，符号扩展，将双字扩展为四字
```

## 3.3 按位运算指令

- `and` ：按位与，将两个操作数的每个位进行与运算，并将结果存储在目的操作数中。
- `or` ：按位或，将两个操作数的每个位进行或运算，并将结果存储在目的操作数中。
- `xor` ：按位异或，将两个操作数的每个位进行异或运算，并将结果存储在目的操作数中。
- `not` ：按位取反，将操作数的每个位进行取反运算，并将结果存储在目的操作数中。
- `test` ：按位与运算，但不存储结果，仅设置标志位。

```assembly
and eax, ebx    ; eax = eax & ebx，设置标志位（ZF、CF等）
or eax, ebx     ; eax = eax | ebx，设置标志位（ZF、CF等）
xor eax, ebx    ; eax = eax ^ ebx，设置标志位（ZF、CF等）
not eax         ; eax = ~eax，设置标志位（ZF、CF等）
test eax, ebx   ; eax & ebx，设置标志位（ZF、CF等），但不存储结果
```

## 3.5 移位指令

- `shl` / `sal` ：逻辑左移，将操作数的每个位向左移动指定的位数，右侧空位补0。
- `shr` ：逻辑右移，将操作数的每个位向右移动指定的位数，左侧空位补0。
- `sar` ：算术右移，将操作数的每个位向右移动指定的位数，左侧空位补符号位（最高位）。

## 3.6 简单条件转移指令

- `jz`/`je` ：当零标志（ZF）为1时，跳转到指定地址。
- `jnz`/`jne` ：当零标志（ZF）为0时，跳转到指定地址。
- `js` ：当符号标志（SF）为1时，跳转到指定地址。
- `jns` ：当符号标志（SF）为0时，跳转到指定地址。
- `jc` ：当进位标志（CF）为1时，跳转到指定地址。
- `jnc` ：当进位标志（CF）为0时，跳转到指定地址。
- `jo` ：当溢出标志（OF）为1时，跳转到指定地址。
- `jno` ：当溢出标志（OF）为0时，跳转到指定地址。
- `jpe` ：当奇偶标志（PF）为1时，跳转到指定地址。
- `jpo` ：当奇偶标志（PF）为0时，跳转到指定地址。

**无符号条件转移指令**

- `ja`/`jnbe` ：当无符号比较结果为大于时，跳转到指定地址。
- `jb`/`jnae` ：当无符号比较结果为小于时，跳转到指定地址。
- `jae`/`jnb` ：当无符号比较结果为大于或等于时，跳转到指定地址。
- `jbe`/`jna` ：当无符号比较结果为小于或等于时，跳转到指定地址。

**有符号条件转移指令**

- `jg`/`jnle` ：当有符号比较结果为大于时，跳转到指定地址。
- `jl`/`jnge` ：当有符号比较结果为小于时，跳转到指定地址。
- `jge`/`jnl` ：当有符号比较结果为大于或等于时，跳转到指定地址。
- `jle`/`jng` ：当有符号比较结果为小于或等于时，跳转到指定地址。

## 3.7 无条件转移指令

- `jmp` ：无条件跳转到指定地址。

```assembly
jmp label    ; 无条件跳转到label标签
label:
    ; some code

lea eax, 0x12345678
jmp eax      ; 无条件跳转到eax寄存器中存储的地址
```

## 3.8 循环控制指令

- `loop` ：将计数寄存器 `ECX` 的值减1，如果结果不为0，则跳转到指定地址。
- `loope` / `loopz` ：将计数寄存器 `ECX` 的值减1，如果结果不为0且零标志（ZF）为1，则跳转到指定地址。
- `loopne` / `loopnz` ：将计数寄存器 `ECX` 的值减1，如果结果不为0且零标志（ZF）为0，则跳转到指定地址。
- `jecxz` ：如果计数寄存器 `ECX` 的值为0，则跳转到指定地址。

```assembly
mov ecx, 5          ; 设置循环计数器

start_label:
    ; some code to execute in the loop

loop start_label    ; loop指令会自动减小ECX的值，到 ecx为0时跳出循环
loope start_label   ; loope指令会自动减小ECX的值，并检查ZF标志，满足条件时跳回start_label
loopne start_label  ; loopne指令会自动减小ECX的值，并检查ZF标志，满足条件时跳回start_label
jecxz start_label   ; 如果ECX已经为0，直接跳转到start_label
```

## 3.9 串操作指令

**串**：指的是一系列连续的内存单元，通常用于存储字符串或数组，表示 “一串字节”、“一串字”等含义。
**esi** 和 **edi** 寄存器在串操作中分别用作源地址和目的地址，**ecx** 寄存器用作计数器，表示要处理的元素数量。
**DF（Direction Flag）**：控制字符串操作的方向。当DF=0时，字符串操作从低地址向高地址进行；当DF=1时，字符串操作从高地址向低地址进行。

**串传送指令**

- `movsb` ：将一个字节从源地址复制到目的地址。
- `movsw` ：将一个字（2字节）从源地址复制到目的地址。
- `movsd` ：将一个双字（4字节）从源地址复制到目的地址。

```assembly
mov esi, src     ; 设置源地址
mov edi, dst     ; 设置目的地址

movsb            ; 将一个字节从[esi]复制到[edi]，并自动更新esi和edi
movsw            ; 将一个字从[esi]复制到[edi]，并自动更新esi和edi
movsd            ; 将一个双字从[esi]复制到[edi]，并自动更新esi和edi
```

**串比较指令**

- `cmpsb` ：比较源地址和目的地址的一个字节。
- `cmpsw` ：比较源地址和目的地址的一个字。
- `cmpsd` ：比较源地址和目的地址的一个双字。

```assembly
; 串比较指令不保存结果，只设置标志位
mov esi, str1    ; 设置源地址
mov edi, str2    ; 设置目的地址

cmpsb            ; 比较[esi]和[edi]的一个字节，设置标志位（ZF、CF等）
cmpsw            ; 比较[esi]和[edi]的一个字，设置标志位（ZF、CF等）
cmpsd            ; 比较[esi]和[edi]的一个双字，设置标志位（ZF、CF等）
```

**串搜索指令**

- `scasb` ：在目的地址中搜索一个字节。
- `scasw` ：在目的地址中搜索一个字。
- `scasd` ：在目的地址中搜索一个双字。

```assembly
mov edi, str      ; 设置目的地址
mov al, target1   ; 设置要搜索的字节
mov ax, target2   ; 设置要搜索的字
mov eax, target3  ; 设置要搜索的双字

scasb            ; 在[edi]中搜索al的值，设置标志位（ZF、CF等）
scasw            ; 在[edi]中搜索ax的值，设置标志位（ZF、CF等）
scasd            ; 在[edi]中搜索eax的值，设置标志位（ZF、CF等）
```

**从原串中取数**

- `lodsb` ：从源地址加载一个字节到累加器寄存器。
- `lodsw` ：从源地址加载一个字到累加器寄存器。
- `lodsd` ：从源地址加载一个双字到累加器寄存器。

```
lodsb             ; AL = [ESI], ESI++/ESI--（取决于DF标志位）
lodsw             ; AX = [ESI], ESI++/ESI--（取决于DF标志位）
lodsd             ; EAX = [ESI], ESI++/ESI--（取决于DF标志位）
```

**将数存入目的串**

- `stosb` ：将累加器寄存器中的一个字节存储到目的地址。
- `stosw` ：将累加器寄存器中的一个字存储到目的地址。
- `stosd` ：将累加器寄存器中的一个双字存储到目的地址。

```assembly
stosb             ; [EDI] = AL, EDI++/EDI--（取决于DF标志位）
stosw             ; [EDI] = AX, EDI++/EDI--（取决于DF标志位）
stosd             ; [EDI] = EAX, EDI++/EDI--（取决于DF标志位）
```

# 4. C 语言程序的机器级表示

## 4.1 存储区：栈

![[img/5/5.png]]

**过程的栈帧**

1. 每个过程P（函数或子程序）都有自己的栈帧，用于保存相关寄存器的值、局部变量、调用其它过程Q的入口参数、过程P的返回地址等信息。
2. 一个程序的所有过程的栈帧构成了程序的栈区。

**与栈相关的寄存器**
- `ESP` ：栈顶指针寄存器，指向当前栈顶的位置。
- `EBP` ：基指针寄存器，指向**当前过程的栈帧**的基地址。
- **当前栈帧的范围**：从 `EBP` 的值到 `ESP` 的值之间的内存区域。

**调用指令call与返回指令ret**

1. `call` 指令会将当前指令的下一条指令的地址（返回地址）压入栈中，然后跳转到被调用过程的入口地址。
2. 被调用过程执行完毕后，使用 `ret` 指令从栈中弹出返回地址，并跳转回调用者继续执行。

![[img/5/6.png]]

## 4.2 过程调用的执行步骤

在过程P调用另一个过程Q时，执行的步骤如下：

1. **调用者准备参数**：P 将参数按照约定的方式（如从右到左）压入栈中，或者通过寄存器传递。这些参数可以被Q访问。
2. **调用指令**：调用者执行 `call` 指令，跳转到被调用过程的入口地址，并将返回地址压入栈中。
3. **被调用过程设置栈帧**：Q 在入口处执行指令，通常会将当前的 `EBP` 压入栈中，然后将 `ESP` 的值复制到 `EBP` 中，建立新的栈帧。这样，Q 就可以通过 `EBP` 来访问自己的参数和局部变量。
4. **被调用过程执行**：Q 执行其代码，可能使用局部变量、参数等。
5. **恢复栈帧**：Q 在返回之前，通常会将 `EBP` 的值恢复到 `ESP` 中，然后弹出之前压入的 `EBP`，恢复调用者的栈帧。
6. **返回指令**：Q 执行 `ret` 指令，从栈中弹出返回地址，并跳转回调用者 P 继续执行。

IA-32对寄存器的保存有一定的约定：
- **调用者保存寄存器**：EAX、ECX、EDX。
- **被调用者保存寄存器**：EBX、ESI、EDI。

当过程P调用过程Q之前，P应该保存EAX、ECX、EDX寄存器的值（如果P需要在调用Q之后继续使用这些寄存器的值），在过程Q执行期间，Q可以自由使用这些寄存器而不需要担心会影响到P的值，等Q执行完毕返回P之后，P可以恢复之前保存的寄存器值继续使用。进入Q的执行期后，对于EBX、ESI、EDI寄存器，Q需要在使用之前保存它们的值，并在返回之前恢复它们的值，以确保调用者P的寄存器状态不被改变。

## 4.3 过程（函数）对于的机器级代码结构

1. **准备阶段**：
    - 形成帧底：`push ebp` 将调用者的基指针压入栈中，保存调用者的栈帧信息;`mov ebp, esp` 将当前的栈顶指针复制到基指针中，建立新的栈帧。
    - 分配局部变量空间：通过调整 `ESP` 的值来为局部变量分配空间，例如 `sub esp, 16` 为局部变量分配16字节的空间。
2. **过程体执行阶段**：
    - 为过程中的非静态局部变量分配空间，并使用 `EBP` 作为基准来访问这些变量。
    - 执行过程的主体代码，可能包括对参数的访问、局部变量的使用、调用其他过程等。
    - 在需要调用其他过程时，按照调用约定准备参数并执行 `call` 指令。
    - 在过程执行完毕后，准备返回值（如果有）并执行返回指令。
3. **结束阶段**：
    - 退栈：`leave` 指令，即 `mov esp, ebp` 和 `pop ebp`，恢复调用者的栈帧。
    - 返回：`ret` 指令，从栈中弹出返回地址并跳转回调用者继续执行。

## 4.4 递归程序的缺点

1. **重复执行**：递归调用过程需要重复执行自身的代码，导致大量的函数调用和返回，增加了程序的执行时间。
2. **栈空间消耗**：每次递归调用都会占用栈空间，如果递归深度过大，可能会导致栈溢出，使程序崩溃。
3. **性能问题**：为了支持过程递归，每个过程都包含了准备阶段和结束阶段的指令，这些指令在每次调用时都会执行，增加了程序的开销。
