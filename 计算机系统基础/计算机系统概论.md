
# 1. 冯·诺依曼结构

## 1.1 最重要的思想："存储程序"

基本思想：任何要计算机完成的工作都要先被编写成程序，然后将程序和原始数据送入主存并启动执行。
  - 一旦程序被启动，计算机能在不需操作人员干预下，自动完成 `逐条取出指令` 和 `执行指令` 的任务。

## 1.2 基本部件

**存储器**：存储程序和数据的地方。
  - `数据` 和 `指令` 都以二进制形式存储在存储器中，形式上没有区别。
  - `指令` 由 `操作码` 和 `地址码` 组成，操作码指定要执行的操作，地址码指定操作数所在的存储器地址。
**控制器**：从存储器中取出指令并执行指令的部件。
**运算器**：执行算术和逻辑运算的部件。
**输入设备**：将数据和程序输入计算机的部件。
**输出设备**：将计算机的结果输出的部件。

## 1.3 工作原理

**程序执行前**

1. 数据和指令都以二进制形式存储在存储器中。
    - 每条指令和每个数据项都有一个唯一的地址。
2. 程序计数器（PC）指向下一条要执行的指令的地址。
    - 初始时，PC指向程序的起始地址。

**程序执行中**

在指令执行过程中，指令和数据被从存储器取到CPU，分别存放在CPU内的对应寄存器中
  - 指令存储在 `指令寄存器IR` 中，数据存储在通用 `寄存器GPRs` 中，实现了指令和数据的区分

基本的工作流程如下：
1. **取指**：控制器从存储器中取出PC指向的指令，并将其存放在指令寄存器IR中。
    - 取指后，PC自动加1，指向下一条指令的地址。
2. **译码**：控制器对指令进行译码，确定要执行的操作（如加法、减法、访问内存）和需要的操作数。
3. **取操作数**：如果指令需要操作数，控制器从存储器中取出操作数并存放在通用寄存器GPRs中。
    - 对于**进行跳转的指令**，控制器会根据指令中的地址码更新PC的值。
4. **执行**：运算器执行指令指定的操作。
5. **写回**：如果指令需要将结果写回存储器，控制器将结果从运算器写回存储器。


# 2. 汇编语言介绍

汇编语言是一种低级编程语言，它直接对应于计算机的机器指令。

我们用 `汇编器` 将汇编语言翻译成机器语言，生成可执行文件，该过程称为 `汇编`。
  - 汇编器将每条汇编指令翻译成对应的机器指令，生成一个二进制文件（如 `.o` 文件）。
  - 链接器将多个 `.o` 文件链接成一个可执行文件（如 `.exe` 文件）。

注：
1. 机器语言和汇编语言都是面向机器结构的语言，故它们统称为机器级语言
2. `反汇编`：将机器语言翻译回汇编语言的过程，通常用于调试和分析程序。

## 2.1 汇编语言的特点

1. **与机器指令一一对应**：每条汇编指令对应一条机器指令，程序员可以直接控制计算机的硬件。
2. **使用助记符**：汇编语言使用助记符（如 `MOV`、`ADD`、`JMP`）来表示操作码，使得编程更容易理解和记忆。
3. **需要手动管理寄存器和内存**：程序员需要手动管理寄存器的使用和内存的分配，这使得编程更复杂，但也提供了更高的性能和控制力。
4. **平台相关**：汇编语言是特定于某种计算机架构的，不同的计算机架构有不同的汇编语言。

## 2.2 高级语言与机器语言的关联

高级语言（如C、Java）通过编译器或解释器转换成机器语言，最终在计算机上执行。
  - 编译器将高级语言代码**翻译成机器语言，生成可执行文件**，执行时启动可执行文件。
  - 解释器**逐行执行**高级语言代码，实时翻译成机器语言。

以GCC编译器的工作流程为例：
```
hello.c（源代码） -> 预处理器 -> hello.i 
-> 编译器 -> hello.s（汇编语言代码） 
-> 汇编器 -> hello.o -> 链接器 -> hello.exe
```
